# WIKI Schema

> 个人知识库的管理规则与操作流程
> 基于 Karpathy LLM Wiki 模式设计

---

## ⚠️ 使用前必读

**本文件是知识库管理的唯一规范，所有入库操作必须严格遵循此文件！**

- **入库前**：必须阅读"三个核心操作 → Ingest"章节
- **页面格式**：必须遵循"页面模板"章节

**违反规范的后果**：知识库结构混乱、日志缺失、无法追溯原文

---

## 核心理念

**传统RAG的问题**：每次查询都从原始文档重新发现知识，零积累。

**Wiki模式的优势**：知识编译一次，持续更新，而非每次重新推导。交叉引用已存在，矛盾已标记，综合已反映所有阅读内容。

## 三个核心操作

### 1. Ingest（摄入）

当有新来源需要入库时：

```
Step 1: 存放原文
- 将原文保存到 raw/ 目录（按类型分articles/reports/papers）
- 原文不可变，作为source of truth

Step 2: 阅读与讨论
- LLM阅读原文，提取关键信息
- 与用户讨论重点、角度、价值判断

Step 3: 判断归属
- 根据核心论点判断主要归属素材库
- 跨主题素材按交叉引用机制处理

Step 4: 写入知识页面
- 在目标素材库创建.md文件
- 严格遵循"页面模板"章节格式

Step 5: 更新索引
- 更新目标素材库的index.md
- 更新总索引INDEX.md

Step 6: 更新相关页面
- 检查是否有相关实体页需要更新
- 添加交叉引用

Step 7: 入库即清
- 删除用户上传/中对应的原文件
- 删除.parsed.md等中间产物
- 用户上传/为纯过渡区，保持为空
```

### 2. Query（查询）

基于wiki回答问题时：

```
Step 1: 检索
- 先读INDEX.md了解全局结构
- 根据问题定位相关素材库
- 读取相关页面

Step 2: 综合
- 基于已有知识页面综合回答
- 引用具体页面作为来源

Step 3: 回填（可选）
- 如果查询产出了有价值的综合分析
- 可以新建知识页面回填进wiki
- 让后续查询可以复用
```

### 3. Lint（检查）

定期健康检查（建议每周一次）：

```
检查项目：
1. 矛盾检测
   - 不同页面之间是否存在观点冲突
   - 标记 [!contradiction] callout

2. 过期检测
   - 是否有内容被新来源推翻
   - 标记 [!stale] callout

3. 孤儿检测
   - 是否有没有被任何页面链接的知识页面
   - 在index.md中标记

4. 缺失检测
   - 是否有重要概念提到但没有独立页面
   - 是否有缺失的交叉引用

5. 完整性检测
   - 是否有页面缺少金句摘录
   - 是否有页面缺少原文备份

输出：Lint报告，记录在LOG.md
```

## 交叉引用机制

当素材跨多个主题时：

1. **物理存储**：根据核心论点判断主要归属，文件只存一份
2. **元数据标注**：保持多主题标签
3. **交叉索引**：在次要归属素材库的index.md建立"相关素材（跨库引用）"栏目
4. **判断原则**：核心论点优先于来源类型

---

## 标签规范

格式：`#场景标签 + #结论标签 + #关键词标签`

要求：3-5个标签，覆盖场景、结论和关键词三个维度。

示例：
- `#AI时代生存法则 #线下体验 #自媒体选题`
- `#AGI社会冲突 #技术变革风险 #非市场风险`

```markdown
- 发现问题数：N
- 处理情况：[概述]
```

## 原文路径字段规范

每个知识页面的 `原文路径` 字段必须填写，取值规则：

| 情况 | 原文路径写法 |
|------|-------------|
| raw/下有原文备份 | `raw/articles/文件名` 或 `raw/reports/文件名` |
| 无独立原文（研报展开） | `无独立原文备份（XX研报展开）` |
| 无独立原文（开源项目） | `无独立原文备份（GitHub开源项目）` |
| 无独立原文（新闻资讯） | `无独立原文备份（新闻资讯）` |
| 晨读记录来源 | `晨读记录（无独立原文备份）` |

**注意**：禁止在原文路径字段使用 `computer://` 协议。

## 根目录整洁规范

**核心原则：文件必须归入文件夹，禁止在根目录随意创建文件。**

根目录下只允许存在**文件夹**和极少数系统级文件（MEMORY / SECRET / USER / SOUL / WIKI / INDEX / LOG / heartbeat_daily / heartbeat_weekly）。任何新文件必须先想好归属哪个文件夹，再创建。
