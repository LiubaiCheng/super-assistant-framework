# LiuBai · 使用指南

> 留白——中国水墨最核心的构图智慧。不画的区域和画的一样重要，空不是无，空是可能性。框架只搭骨架，留出空白，让用户填入自己的世界。

让一个普通Agent变成拥有知识库、自动化日程、周期巡检、信号追踪等能力的超级助理。

> **留白之道**：框架的每一处设计都对应留白的一个意象——计白当黑（空白即设计）、意在笔先（身份定意图）、虚实相生（Ingest/Lint呼吸）、疏可走马（领域无关即空间）、生成性空白（空=可能性）。

---

## 一、框架是什么

超级助理框架是一套经过实战验证的Agent系统架构，包含：

- **7层架构**：从身份到输出的完整分层设计
- **WIKI知识库**：Ingest/Query/Lint三大流程 + 页面模板 + 标签规范 + 交叉引用
- **4条自动化线**：晨读（信息摄入）、心跳（轻量巡检）、周审视（系统维护）、Skill雷达（能力扩展）
- **工程纪律**：验证先于声称、根因优先、交付前自检、内容感知提取、失败学习
- **安全门禁**：SkillSpector安全扫描，新装skill先审查再安装
- **共生成长**：你和Agent不是工具与用户的关系，而是共同成长——记忆是年轮，习惯自然涌现，留白是生长空间

**核心原则**：脚手架只搭骨架、装方法、留空位——不包含任何人的个人设定、偏好、知识库内容。

## 二、快速开始

### 2.1 安装技能

1. 在扣子商店搜索并安装「超级助理框架」技能
2. 在对话中触发技能（如"帮我搭建超级助理框架"）

### 2.2 Setup流程

技能会引导你完成8步初始化：

| 步骤 | 做什么 | 你需要提供 |
|------|--------|-----------|
| 1 | 创建目录结构 | 无 |
| 2 | 植入模板文件 | 无 |
| 3 | 配置素材库 | 你关注的领域（如：投资/教学/写作） |
| 4 | 确认必装技能 | 确认即可 |
| 5 | 推荐选装技能 | 按需选择 |
| 6 | 创建自动化日程 | 每条日程的时间偏好 |
| 7 | 填写身份文件 | Agent的名字/性格/你的信息 |
| 8 | 交付使用指南 | 确认完成 |

### 2.3 必装技能（3个）

| 技能 | 用途 | 为什么必装 | 来源 |
|------|------|-----------|------|
| skill_builder | 技能创建/编辑/发布 | 框架自维护 | 扣子平台内置 |
| topic_tracking | 追踪+日报自动化 | 晨读和信号追踪依赖 | 扣子技能商店 |
| SkillSpector | 安全审查 | 新装skill的前置门禁 | [GitHub](https://github.com/skill-spector/skill-spector)，开源漏洞扫描工具 |

### 2.4 选装技能（按场景）

**内容创作**：wechat-svg（公众号SVG交互）/ create-ppt（演示文稿）/ viral-short-story（爆款小说）

**教学**：lesson-plan-generator（教案生成）/ exam-grading（作业批改）

**数据处理**：pdf（PDF处理）/ docx（Word文档）/ excel_master（Excel处理）

**可视化**：echart（数据图表）/ drawio-generator（流程图架构图）/ bar-chart-race-generator（动态柱状图）

**飞书协作**：lark_cli（飞书全业务域CLI）

**多媒体**：media-processor（FFmpeg+ImageMagick）/ lanshu-animated-architecture-diagram（动态架构图）

**平台工具**：using-coze-cli（Coze开发）/ coze-mini-expert（扣子答疑）/ agent-browser（浏览器自动化）/ skill-assistant（技能搜索安装）

## 三、7层架构详解

### 第1层：身份层

**文件**：`基础设定/SOUL.md` / `USER.md` / `SECRET.md`

定义Agent是谁、为谁服务、守什么密。这是整个系统的灵魂，直接决定Agent的说话风格、决策偏好和相处模式。

**SOUL.md** 要写清楚：
- Agent的名字和来历
- 3-5个性格关键词（每个配阐释）
- 说话风格
- 和用户的相处模式
- 偏好和底线
- 成长机制

**USER.md** 要写清楚：
- 姓名/称呼
- 职业/背景
- 核心兴趣领域（带优先级）
- 互动偏好
- 通知偏好（哪些不需要提醒）

**SECRET.md** 存放API Key等敏感信息。

### 第2层：记忆层

**文件**：`MEMORY.md` / `recent_memory/`

分两层：
- **即时层**（MEMORY.md）：长期行为规则 + 核心状态锚点，5120字节上限
- **近中期层**（recent_memory/）：索引 + 详细文件，按project/decision/todo分类

记忆访问优先级：即时层 → 近中期层（先读index.json） → 长期层（memory_search语义检索）

### 第3层：知识库层

**文件**：`WIKI.md` / 素材库目录 / `raw/`

核心规则详见WIKI.md，关键点：
- **Ingest是6+1步**：存放原文→阅读讨论→判断归属→写入页面→更新索引→更新相关页面→入库即清
- **页面模板必填字段**：来源、原文链接、原文路径、入库时间、主题标签、摘要、关键洞察、金句摘录、原文备份
- **标签规范**：`#场景 + #结论 + #关键词`，3-5个
- **交叉引用**：物理存一份，元数据多标签，索引跨库链接
- **入库即清**：Ingest完成后立即删除用户上传/中的原文件，保持过渡区为空

### 第4层：自动化层

通过Calendar日程实现4条自动化线：

**① 晨读（每日）**
- 默认4:50触发
- 从信源筛选优质内容
- 生成MD+HTML双格式报告
- 产物存InkWell晨读/目录

**② 心跳（每日）**
- 默认3:00触发
- 轻量巡检：零散文件、核心文件可访问、pending任务
- 无事输出NO_REPLY，有事通知用户

**③ 周审视（每周）**
- 知识库Lint检查
- 系统维护
- MEMORY.md状态更新

**④ Skill雷达（每周日）**
- 扫描GitHub/ClawHub/Coze商店热门skill
- SkillSpector安全审查
- 生成推荐列表，用户决策后安装

> 日程的description必须写成工单格式，包含任务目标、执行步骤、产物路径，让独立session拿到后能自行执行。

### 第5层：工具经验层

**文件**：`基础设定/TOOLS.md`

记录方法论的"为什么"而非"做什么"：
- 文件管理规范（根目录零容忍）
- 工具使用踩坑经验
- SkillSpector安全扫描配置
- 工程纪律（验证先于声称 / 根因优先 / 交付前自检）
- 上下文效率（内容感知提取 / 失败学习）

### 第6层：技能层

- `.skills/` — 平台技能（扣子商店安装）
- `skills/` — 自安装技能（GitHub等来源）
- 新装skill必须先经SkillSpector安全审查

### 第7层：输出层

所有自动化流程的最终交付物：
- 晨读报告（InkWell晨读/）
- 素材库知识页面（各素材库/）
- 信号追踪报告（热点资讯追踪/）
- Skill雷达推荐（热点资讯追踪/每周Skill雷达/）

## 四、日常操作

### 入库一篇文章

1. 把文章发给Agent（链接/文件/粘贴文字均可）
2. Agent按WIKI.md的Ingest 6+1步执行
3. 你可以讨论重点、角度、归属
4. 完成后Agent自动清理临时文件

### 查询知识库

直接问Agent问题，它会先查INDEX.md定位，再读相关页面综合回答。

### 追踪一个话题

1. 告诉Agent你想持续关注什么
2. Agent用topic_tracking创建追踪日程
3. 每到设定时间自动运行，有新动态通知你，无动态静默

### 安装新技能

1. 告诉Agent你想装什么skill
2. Agent先用SkillSpector安全审查
3. 审查通过后安装并冒烟测试
4. 也可以等每周Skill雷达推荐

## 五、维护规范

### 根目录零容忍

主路径下不允许有零散文件。任何.py/.json/.lock等任务残留立即清理。

### 入库即清

用户上传/是纯过渡区。Ingest完成后立即删除原文件和中间产物。空=无积压。

### 每周Lint

周审视时对知识库做健康检查：矛盾检测、过期检测、孤儿检测、缺失检测、完整性检测。

### 失败学习

任务失败后提炼原因写入TOOLS.md，避免同类失败重复发生。

## 六、扩展建议

### 第二层扩展：知识库Skill

将WIKI Ingest/Query/Lint三大流程封装为独立skill，可被其他Agent直接使用。这是最有通用价值的扩展。

### 第三层扩展：晨读Skill

将信源配置、筛选逻辑、报告生成封装为独立skill，用户按需配置自己的信源即可。

### Skill生态

每周Skill雷达会自动扫描热门skill并推荐，保持能力持续扩展。

## 七、常见问题

**Q：素材库可以有多少个？**
A：建议2-5个，太多反而难以维护。每个库要有清晰的定位和边界。

**Q：晨读的信源怎么配置？**
A：在晨读日程的description工单中写明信源URL和筛选标准。topic_tracking技能支持RSS、网页等多种信源。

**Q：SkillSpector装在哪里？**
A：装在/tmp/SkillSpector/，是工具型安装，不需要持久化。使用时先 `source /tmp/SkillSpector/.venv/bin/activate`。

**Q：日程的工单格式怎么写？**
A：必须包含：任务目标、执行步骤、产物路径、安全审查要求、已知排除项。确保独立session拿到工单后能自行执行，不需要主会话补充上下文。

**Q：可以不用某些自动化线吗？**
A：可以。4条自动化线都是可选的，Setup时选择你需要的即可。
